  <action>
Replace lines 208-231 (the xmlContent-based product counting block) with a single raw SQL query against product_registry.

The product_registry table tracks products from RECEIVED invoices, keyed by company_id. However, it does not directly store the sender CNPJ per product. Instead, we need to use the invoice_item_tax table which has both invoice_id and product fields, joined with Invoice to get senderCnpj.

Replace the existing block with:

```sql
SELECT i."senderCnpj" as cnpj, COUNT(DISTINCT CONCAT(it.product_code, '::', it.product_name, '::', it.product_unit)) as cnt
FROM invoice_item_tax it
INNER JOIN "Invoice" i ON i.id = it.invoice_id
WHERE it.company_id = $1
  AND i."type" = 'NFE'
  AND i."direction" = 'received'
  AND i."senderCnpj" = ANY($2)
GROUP BY i."senderCnpj"
```

Use `prisma.$queryRawUnsafe` with the company.id and paginatedCnpjs array as parameters. Build the priceItemCountMap from the result rows. Wrap in try/catch to handle case where invoice_item_tax table does not exist yet (fallback to 0 counts).

Remove the `xmlContent` import entirely from this code path. The `invoiceXmls` findMany and all the regex parsing inside the for-loop should be deleted.
