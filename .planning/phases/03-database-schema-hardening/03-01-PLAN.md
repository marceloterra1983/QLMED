# Phase 3 Plan 1: Database Schema Hardening

## Objective

Add missing indexes to Invoice model and create @@ignore model stubs for 6 shadow tables that exist in PostgreSQL but have no Prisma representation.

## Tasks

### Task 1: Add Invoice indexes (DB-01)
- Add `@@index([recipientCnpj])` to Invoice model
- Add `@@index([companyId, recipientCnpj])` to Invoice model
- These optimize queries that filter by recipient CNPJ

### Task 2: Add shadow table stubs (DB-02)
- Read CREATE TABLE from each lib file to understand column structure
- Add `@@ignore` model stubs for:
  1. `product_registry` (src/lib/product-registry-store.ts)
  2. `stock_entry` (src/lib/stock-entry-store.ts)
  3. `nfe_entry_item` (src/lib/stock-entry-store.ts)
  4. `product_settings_catalog` (src/lib/product-settings-catalog.ts)
  5. `cnpj_cache` (src/lib/cnpj-lookup.ts)
  6. `cnpj_monitoring` (src/lib/cnpj-monitor.ts)

## Verification

- `npx prisma validate` passes
- `npm run build` passes
- No `prisma db push` executed (shared DB)

## Success Criteria

- Both indexes present in Invoice model
- All 6 shadow tables have model stubs with correct column mappings and @@ignore
